# Page Transitions Architecture

## Page Structure (outside Barba container)

```
div.page-wrapper [data-barba="wrapper"]
  div (global component wrapper)
    div.banner_component        ← scrolls away naturally
    nav.navbar_component
      div.mega-nav              ← sticky, top: 0
    main.main-wrapper [data-barba="container"]
      section...
```

Banner and navbar sit **outside** the Barba container. This is critical — they persist across transitions and must be managed manually.

## Crossfade Transition Flow (sync: true)

### Phase 1 — `beforeEnter` (before any animation)

1. Stop Lenis
2. Save `window.scrollY` as `savedScrollY`
3. **Pin old container** at its visual position (`position: fixed; top: getBoundingClientRect().top`) — prevents it from moving when we scroll
4. `window.scrollTo(0, 0)` — invisible because old container is pinned
5. **Shift banner/nav up** with `gsap.set({ y: -shift })` where `shift = Math.min(savedScrollY, bannerH)` — maintains their "scrolled-away" visual state
6. Read `navBottom` from `.mega-nav.getBoundingClientRect().bottom` — now reflects the shifted visual position
7. Set new container to `position: fixed; top: navBottom`

### Phase 2 — Crossfade (leave + enter run simultaneously)

- Old container fades out (`autoAlpha: 0`, 0.6s) — pinned in place, no movement
- New container fades in (`autoAlpha: 0 → 1`) — fixed at top, showing page from top
- **Nothing visual changes** regarding scroll or banner position during this phase

### Phase 3 — `resetPage` (after crossfade, old container removed)

1. Animate banner/nav from `y: -shift` to `y: 0` (banner slides into view, 0.3s)
2. Simultaneously animate container `top` from `navBottom` to `bannerH + navH` (tracks nav moving down)
3. On complete: `clearProps` on banner/nav transforms and container positioning — no jump because `top` already matches natural-flow position
4. `ScrollTrigger.refresh()`, restart Lenis

## Key Principle

> All scroll resets and position compensation happen **invisibly** in `beforeEnter` (using fixed positioning and transform offsets). The banner reveal animation happens **after** the crossfade completes. Never call `window.scrollTo()` or animate banner/nav while both pages are visible.

## Anchor Links & Barba Prevention

### Problem

Barba intercepts all `<a>` clicks. Anchor links (`#heading-id`) — including those generated by Finsweet TOC — trigger unwanted page transitions.

### Why `prevent` callback alone fails

- After Barba navigation, generated links may have full URLs (`/path#hash`) not just `#hash`
- Webflow URLs have inconsistent trailing slashes (`/path` vs `/path/`)
- Barba's `prevent` callback isn't always reliable for edge cases

### Solution: Capture-phase click handler (`lenis.js`)

A `document.addEventListener('click', handler, true)` (capture phase) fires **before** Barba's bubble-phase handler. It:

1. Checks `link.hash` exists and `link.pathname` matches current page (with trailing-slash normalization)
2. Verifies target element exists in DOM (avoids breaking Webflow tabs/accordions that use `#`)
3. Calls `e.preventDefault()` + `e.stopPropagation()` — Barba never sees the event
4. Uses `lenis.scrollTo(target, { offset: -navHeight })` for smooth scrolling

The Barba `prevent` callback is kept as a secondary safeguard.

## Finsweet Attributes Reinit

The site uses **Finsweet Attributes v2** (`@finsweet/attributes@2`).

- `reinitFsAttributes()` calls `.restart()` on every loaded module in `window.FinsweetAttributes.modules`
- Each `.restart()` is wrapped in try-catch — some modules may not support it
- **Do NOT call `FinsweetAttributes.load(key)`** for modules not already loaded — it throws and breaks the entire `afterEnter` hook, halting all Barba navigation
- The v1 fallback (`window.fsAttributes.push`) is kept but unlikely to be used

## Common Pitfalls

1. **Never scroll or animate banner/nav during crossfade** — both pages are visible, any position change is jarring
2. **`FinsweetAttributes.load()` can throw** — never call it without a try-catch or existence check
3. **Trailing slashes matter** — always normalize with `.replace(/\/$/, '')` when comparing pathnames
4. **Barba's `prevent` fires in bubble phase** — use capture-phase handlers to reliably intercept clicks before Barba
5. **`getBoundingClientRect()` includes CSS transforms** — useful for reading visual positions after `gsap.set({ y: ... })`
6. **The `leavingContainer` variable** — set in `beforeEnter`, used for pinning; only set when `current !== next` container
